# 🧩 문제

## [네트워크](https://school.programmers.co.kr/learn/courses/30/lessons/43162)

### 💡 아이디어 / 접근법

1. n 만큼의 visited 배열 생성
2. dfs 탐색 시작, 방문하지 않았고, 컴퓨터가 연결되어 있다면
3. 재귀 호출
4. 재귀 호출이 다 끝나면 카운팅

### 🤔 후기

2차원 배열 + dfs 문제로 다음 들어갈 조건을 생각 잘 하면 어려운 문제는 아니었음

**2.** [여행경로](https://school.programmers.co.kr/learn/courses/30/lessons/43164)

### 💡 아이디어 / 접근법

1. 티켓의 길이만큼의 vistied 배열 생성
2. dfs로 끝까지 내려갔을 경우 정답에 루트 추가
3. 아직 방문하지 않고 마지막에 추가한 값과 현재 시작값이 같다면
4. 현재 끝값과 지금까지의 루트를 넣어주기
5. 정렬 후 첫번째 요소 출력

### 🤔 후기

지금 현재값과 이전 경로의 마지막 값을 비교하는 로직만 생각했으면 기본적인 dfs 문제였다고 생각함
