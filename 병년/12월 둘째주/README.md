# 🧩 문제

## [최소 힙](https://www.acmicpc.net/problem/1927)

### 💡 아이디어 / 접근법

1. 최소 힙 (이것도 개빡셈)
2. 0들어오면 pop
3. 아니라면 push

### 🤔 후기

이 문제 또한 JS 가 아니었다면 쉽게 풀렸을 문제였으나.. JS 이기 때문에 직접 다 구현을 해줘야 했다 그래서 저번에 풀었던 최대 힙에서 조건만 바꿔서 돌렸더니 성공 해버림..날먹함..

## [절대값 힙](https://www.acmicpc.net/problem/11286)

### 💡 아이디어 / 접근법

1. 절대값 힙 (제일 빡셈)
2. 0들어오면 pop
3. 아니라면 push

### 🤔 후기

얘는 기존 최소힙에서 두 원소의 절대값을 비교한 뒤 진행해줘야 했고 또한, 왼쪽 요소와 부모 요소, 오른쪽 요소와 부모 요소를 비교하여 최소 값을 찾아야 하는 문제였다.. 솔직히 풀다가 막혀서 GPT 한테 물어봐씀..

## [전력망을 둘로 나누기](https://school.programmers.co.kr/learn/courses/30/lessons/86971)

### 💡 아이디어 / 접근법

1. BFS 로 그래프 탐색
2. 이어진 노드가 현재 끊어진 노드라면 건너뛰고 방문하지 않았다면 계속 진행 << 이게 포인트
3. 양방향 노드기 때문에 a, b b, a 모두 탐색해서 둘의 차이를 절대 값으로 반환하고 최소값 비교

### 🤔 후기

오랜만에 다시 프로그래머스 문제를 풀러왔는데 문제 자체는 어렵지는 않았음. 근데 뭔가 처음에 보면 이걸 어떻게 BFS로 풀 수 있을까 고민을 많이 해봤던 문제였음. 특히 어떻게 끊어진 걸 나타낼 수 있을 지에 대해 고민을 했는데 사실 이어진 노드들 중 끊고자 하는 노드를 건너 뛰면 되는 거였음 ㅋ

## [단어 변환](https://school.programmers.co.kr/learn/courses/30/lessons/43163)

### 💡 아이디어 / 접근법

1. 두 단어 체크해서 단어 내부의 단어가 몇 개나 다른 지 체크
2. BFS, DFS 로 탐색 진행
3. 현재 단어가 target 이라면 depth 반환

### 🤔 후기

음 이게 프로그래머스 기준 LV3 문제였고 기존에 DFS 로 풀었던 적이 있어서 뭐 그렇게 어렵진 않았고 실제 문제를 푸는데도 한 30분 정도 쓴 것 같음 이것으로 이번주 문제 끝!
