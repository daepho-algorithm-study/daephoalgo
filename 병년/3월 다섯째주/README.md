# 🧩 문제

## [저울](https://www.acmicpc.net/problem/10159)

### 💡 아이디어 / 접근법

### 플로이드 워셜 풀이

1. 내 위치 0으로 초기화 
2. 입력 값에 따라 거리 배열에 dist[a - 1][b - 1] = 1로 체크
3. 플로이드 워셜을 돌려서 각 노드별로 노드끼리의 상하관계 판별
4. 플로이드 워셜을 통해 만들어진 리스트 탐색
5. 상하관계를 판별할 수 없는 무게추를 출력하기 위해 cnt 변수 정의
6. dist[i][j] or dist[j][i] 가 infinity 라면 i와 j의 상하관계를 판별할 수 없기 때문에 cnt++
7. 결과 배열에 cnt 저장


### 🤔 후기

과거 dfs를 통해 정방향 그래프와 역방향 그래프의 방문 체크로 동일한 로직을 구현했던 적이 있는데 결국 상하관계 판별 문제이기 때문에 플로이드 워셜로 풀 수 있을 것 같아서 그렇게 풀어봄

## [면접 보는 승범이네](https://www.acmicpc.net/problem/17835)

### 💡 아이디어 / 접근법

1. 도로 정보와 가중치 그래프에 입력 최적화를 위해 역방향으로 그래프 만들기 (모든 도시에 대해 면접장의 수를 구하면 N + 1만큼 다익스트라를 수행해야 하지만 면접장에서 도시를 찾는 거로 바꾸면 면접장을 넣어놓고 다익스트라 한 번만 하면 되기 때문에)
2. 거리 배열 infinity로 초기화
3. 다익수뚜라 실행
4. 최소값 힙으로 큐를 만들고 면접장의 위치와 초기 비용 삽입
5. 현재 위치와 비용 큐에서 뽑아 오기
6. 지금까지 저장된 현재 위치의 비용이 방금 뽑은 비용보다 작다면 넘어가기
7. 현재 위치에 저장된 도로의 정보를 next, 비용 순서로 봅아오기
8. 다음 비용을 지금 뽑은 비용에 다음의 비용을 더한 값으로 설정
9. 그 값이 지금 저장되어 있는 다음 노드에 대한 비용보다 작다면
10. 바꿔주고 큐에 집어 넣기
11. mx 값 뽑고 dist 배옇 순회하면서 mx값이랑 똑같은 값 넣어주고 그 중 가장 낮은 index 출력

### 🤔 후기

이 문제는 보자마자 다익스트라로 풀어야겠다고 생각했음 왜 why? 저번에 문제 풀 때 생각해보니까 모든 노드에 대해 동일한 도착지점이 정해져있는 경우 프로이드 와샬 보다는 다익스트라가 더 좋기 때문임 당연하게도 시간때문에 그래서 다익스트라로 풀었고 처음에는 노드에 대해 면접장의 거리를 뽑는 걸로 로직을 수행했는데 시간초과가 나더라 그래서 그래프를 면접장 기준으로 뒤집어 봤고 그렇게 하니까 다익스트라를 반복문에서 실행시키지 않아도 되더라고

## [파티](https://www.acmicpc.net/problem/1238)

### 💡 아이디어 / 접근법

1. 도로 정보와 가중치 그래프에 입력, 뒤집은 방향도 추가(다시 돌아오는 경우를 위해서)
2. 데잌수투라 실행
4. 최소값 힙으로 큐를 만들고 면접장의 위치와 초기 비용 삽입
5. 현재 위치와 비용 큐에서 뽑아 오기
6. 지금까지 저장된 현재 위치의 비용이 방금 뽑은 비용보다 작다면 넘어가기
7. 현재 위치에 저장된 도로의 정보를 next, 비용 순서로 봅아오기
8. 다음 비용을 지금 뽑은 비용에 다음의 비용을 더한 값으로 설정
9. 그 값이 지금 저장되어 있는 다음 노드에 대한 비용보다 작다면
10. 바꿔주고 큐에 집어 넣기
11. 정방향 그래프로 다익스트라 돌려서 i -> X 거리 뽑아오고 역방향 그래프로 다익스트라 돌려서 X -> i 거리 배열 뽑아오기
12. 그 둘을 순회하면서 더하고 그 중 최대값이 정답임

### 🤔 후기

이것도 특정 노드 거쳐서 다시 돌아오는 문제라서 플로이드 워셜을 먼저 떠올렸고 실제로 그렇게 풀어봤는데 시간 초과가 나더라 그래서 다익스트라로 바꿔서 풀어봤음 근데 단방향 그래프이기 때문에 반대 방향에서 돌아오는 로직은 역방향 그래프를 하나 추가해서 카바해줬음 그러니까 풀리더라 근데 문제 게시판 보면 Java는 플로이드로 통과하는 것 같던데 나중에 풀어줄사람 ㅋ
