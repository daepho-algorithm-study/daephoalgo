# 🧩 문제

## [벽 부수고 이동하기](https://www.acmicpc.net/problem/2206)

### 💡 아이디어 / 접근법

1. 델타 탐색을 위한 배열 생성
2. 다익스트라 알고리즘을 위한 dist 배열 생성
3. 부쉈는 지 안 부쉈는 지 확인하기 위해 3차원 배열로 생성
4. 초기값 1로 설정 및 우선순위 큐에 삽입
5. 다익스트라 수행
6. 4방향 탐색을 돌리면서 다음 노드의 값이 0이라면
7. 벽을 부술 필요가 없기 때문에 단순하게 비용 체크 및 비용 재할당 후 우선순위 큐에 삽입
8. 그게 아니라 다음 노드의 값이 1이라면
9. 벽을 부순 횟수가 1회 미만이라면 
10. 벽을 부수고 부쉈을 때 비용이 지금 비용보다 더 크다면
11. 지금 비용을 할당하고 우선순위 큐에 삽입
12. 마지막에 벽을 한 번도 안부수고 목적지에 간 경우와 벽을 한 번 부수고 목적지에 간 경우 중 최소값 출력

### 🤔 후기

벽을 부수고 안부수고를 기준으로 다익스트라를 수행하고 해당 값에서 최소값을 찾는 문제였는데 처음에는 cost에 벽 부수고 안 부수고 같이 체크하려고 해서 값이 계속 이상하게 나왔음 그래서 3차원 배열로 만들어서 벽을 부술지 안부술지 분기를 나눠서 다익스트라를 진행했고 거기서 최소값을 찾는 걸로 바꿨더니 성공했음

## [벽 부수고 이동하기 2](https://www.acmicpc.net/problem/14442)

### 💡 아이디어 / 접근법
(다익스트라)
1. 벽 부수고 이동하기 1에서 부술 수 있는 벽의 개수 만큼 dist 배열에 추가해서 다익스트라를 수행
2. 결과값 나오는 걸 확인했으나 시간초과

(bfs)
1. 3차원 dist 배열생성(vistied)
2. 초기 값 할당 후 bfs 실행
3. 목적지에 도착하면 depth 반환
4. 델타 탐색 진행하면서 다음 값이 벽을 부수지 않고 이동할 수 있을 때
5. 아직 방문하지 않았다면 방문 체크 후 depth + 1후 queue에 삽입
6. 다음 값이 부숴야 이동할 수 있을 때
7. 벽을 부순 회수가 K 보다 작다면
8. 벽을 부수고 해당 값이 아직 방문하지 않았다면
9. 방문 체크 후 queue에 삽입

### 🤔 후기
 
처음에는 벽 부수고 이동하기 1에서 K 만큼 dist 배열에 추가하면 될 것이라 생각해서 dist 배열에서 length를 K + 1 로 변경 후 실행 답은 나오지만 시간초과로 실패했음 그래서 해당 문제는 다익스트라가 아닌 일반 BFS 로 돌려야 함을 깨닫고 BFS 로 바꿔서 제출하니 성공함 굉장히 어려웠음..

## [최소비용 구하기2](https://www.acmicpc.net/problem/11779)

### 💡 아이디어 / 접근법

1. 그래프 파싱해서 가중치 삽입
2. 역추적 배열 생성 후 다익스트라 진행
3. 현재 비용과 노드 값 뽑고 다음 노드 탐색
4. 다음 거리의 비용이 지금 뽑은 비용값 보다 크다면
5. 재할당 후 역추적 배열[next] 에 현재 노드 값 집어 넣고 역추적 가능하도록 함
6. while 문을 통해 prev 배열 순회하면서 경로 복원
7. 복원한 경로 뒤집어서 출력

### 🤔 후기
 
기본 다익스트라 문제에 역추적이 포함된 알고리즘으로 언제 역추적 배열에 값을 할당할 지만 판단하면 그렇게 어려운 문제는 아니었음
