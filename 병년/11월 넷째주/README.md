# 🧩 문제

## [최소비용 구하기](https://www.acmicpc.net/problem/1916)

### 💡 아이디어 / 접근법

1. 단방향 노드로 입력값 파싱하기
2. 시작 좌표 기준으로 다익스트라 수행
3. 다익스트라 함수를 통해 각 좌표에 해당하는 최소한의 거리를 dist 배열에 담기
4. 도착 지점의 거리 값 반환

### 🤔 후기

다익스트라 기본적인 문제였고, 시작 좌표 값과 끝 좌표 값이 주어지기 때문에 모든 좌표를 다 돌려볼 필요가 없었음 또, 일반 배열로 다익스트라를 돌려도 시간초과가 나지 않았는데, 입력값이 좀 더 커지거나 시간제한이 더 있다면 힙으로 구현하는 방안도 생각해봐야 할 것 같음 다익스트라 기본 문제였는데도 구글링을 좀 하면서 했음.

## [서강그라운드](https://www.acmicpc.net/problem/14938)

### 💡 아이디어 / 접근법

1. 양방향 노드로 입력값 파싱하기
2. 반복문을 통한 좌표별로 다익스트라 수행
3. 다익스트라 수행 후 정점의 거리가 수색 범위보다 작은 경우 sm에 먹을 수 있는 아이템의 개수 합산
4. 결과 값 계속 최댓값으로 계속 갱신해주기

### 🤔 후기

이 문제 또한 다익스트라 기본 문제였고, 이 문제가 오히려 모든 좌표에 대해 검사하는 문제라서 더 일반적인 상황에서 많이 쓰일법한 다익스트라 문제였음. 첫 번째 문제를 풀고 나서 풀었더니 크게 어려움 없이 풀었던 것 같음.

## [양치기 꿍](https://www.acmicpc.net/problem/3187)

### 💡 아이디어 / 접근법

1. 델타 탐색 + dfs 
2. dfs 내부에서 양의 개수와 늑대의 개수 탐색
3. 반복문 + dfs 를 통해 울타리 내부의 늑대와 양의 개수를 판별하고 결과 값에 어떤 쪽이 더 많이 살아남았는 지 누적

### 🤔 후기

델타와 dfs를 다룰 줄 안다면 기본적으로 쉬운 문제였음.
