# 🧩 문제

## [나무 자르기](https://www.acmicpc.net/problem/2805)

### 💡 아이디어 / 접근법

1. 이진 탐색으로 절단기의 높이를 설정하기
2. 자른 나무의 수가 M 과 크거나 같을 경우 정답 후보에 넣고 절단기의 높이를 높이기
3. 아닌 경우에는 절단기의 높이를 낮추기

### 🤔 후기

기본적인 이분탐색 문제로 풀었고 정답 갱신 조건 하나만 추가된 문제였다 이분탐색 다시 해보려고 하니까 어렵다 그때도 이분탐색은 많이 안풀었던 거 같아서 이번에 좀 많이 풀어봐야지

## [숫자 카드](https://www.acmicpc.net/problem/10815)

### 💡 아이디어 / 접근법

1. 이진 탐색으로 전체 카드중 상근이가 가지고 있는 카드의 경우 보기
2. 존재하면 result[i] 번째를 1로 바꿔주기
3. 더 큰경우에는 left 값 늘려주고 아니라면 right 값 줄여주기

### 🤔 후기

이분탐색 대표적인 문제 중 하나라고 생각하는데, 아직 이분탐색에서 어떤 값을 기준점으로 두고 문제를 풀어야 할 지 감이 잘 오지 않아서 해맸음 어렵다..

## [랜선 자르기](https://www.acmicpc.net/problem/1654)

### 💡 아이디어 / 접근법

1. 이진 탐색으로 기준이 될 길이 찾기
2. 현재값에서 arr의 원소를 나누고 정수부분만 cnt에 누적으로 더해주기
3. cnt 가 M 보다 크다면 answer 재할당 후 left 값 늘려서 더 경우가 존재하는 지 탐색
4. 아니라면 right 값 줄여서 탐색

### 🤔 후기

나무 자르기 문제랑 거의 동일한 문제였고 다른 점은 floor 를 통해 num / mid 값을 누적해주는 것이었음. 쉽게 풀었던 거 같음

## [공유기 설치](https://www.acmicpc.net/problem/2110)

### 💡 아이디어 / 접근법

1. 먼저 제일 마지막에 설치한 값(last) arr[0] 으로 시작하여 처음 공유기 설치 하기
2. arr의 원소에서 last 를 뺀 값이 mid 보다 크다면 cnt 를 누적하고 last 를 재 할당
3. cnt 의 값이 C보다 크거나 같을 경우 공유기를 놓을 수 있는 경우의 수가 되기 때문에 result에 재 할당 후 더 놓을 수 있는 경우가 존재하는 지 탐색
4. 아니라면 right 값 줄여서 탐색

### 🤔 후기

진짜 너무 어려웠음. 어떤 값으로 기준을 두고 줄이고 늘릴 지 생각하는 게 좀 많이 어려웠던 거 같음 이분탐색에서 해당 부분이 되게 중요하다고 생각하는데 엄청 어려웠던 거 같음..

## [예산](https://www.acmicpc.net/problem/2512)

### 💡 아이디어 / 접근법

1. 기본적인 이분탐색 문제로 접근
2. 원소의 값이 mid 보다 크다면 누적합에 mid 누적 아니라면 원소 값 누적
3. 누적합이 limit 보다 작다면 결과 재할당하고 범위 늘려서 탐색
4. 아니라면 right 값 줄여서 탐색

### 🤔 후기

이거는 되게 금방 풀었음 생각한 그대로 답이 나와서 쉬웠음 굿

## [두 용액](https://www.acmicpc.net/problem/2470)

### 💡 아이디어 / 접근법

1. 투 포인터로 시작과 끝 값 정의
2. 포인터가 가리키는 원소의 합을 구하고 최소값 재할당
3. 만약 합이 0보다 작으면 시작값을 높여서 더 찾을 수 있는 지 판별 반대로 크다면 끝 값을 낮춰서 판별
4. 배열에 저장후 출력

### 🤔 후기

옛날에 풀었던 문제인데 문제 보고 그 기억이 나서 아이디어 금방 찾고 풀었음!
