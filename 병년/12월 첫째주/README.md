# 🧩 문제

## [농장 관리](https://www.acmicpc.net/problem/1245)

### 💡 아이디어 / 접근법

1. DFS 혹은 BFS 로 산봉우리 탐색
2. 만약 다음 좌표의 높이가 현재 위치보다 크다면 탐색하지 않음
3. 만약 다음 좌표가 현재 좌표와 같다면 탐색 진행(어차피 작은 좌표는 볼 필요 없고 나랑 같은 높이만 판별하면 됨)
4. 탐색 후 결과 갱신

### 🤔 후기

문제 자체는 델타 + DFS || BFS 문제인데 이제 탐색 조건을 하나 더 걸어줘야 했음 내 높이보다 다음 높이가 크면 탐색을 진행하면 안되고 높이가 낮은 경우 어차피 볼 필요가 없음 그래서 사실상 현재 좌표에서 가장 높은 높이를 찾고 그 높이가 인접한 곳에 또 존재하면 그게 가장 높은 산봉우리가 되는 것임 그래서 결론적으로 가장 높은 높이를 찾고 혹시 주변에 동일한 높이를 가진 산봉우리가 있는 지를 판별하는 문제였음.

## [가장 가까운 공통 조상](https://www.acmicpc.net/problem/3584)

### 💡 아이디어 / 접근법

1. 조상을 저장하기 위한 set 선언
2. A 의 조상 저장
3. B 의 조상 탐색 하면서 Set에 존재하면 배열에 저장

### 🤔 후기

음 어려운 문제는 아니었음 단순히 dfs 2번 돌려서 조상 찾고 조상 존재하면 반환하면 끝인 문제 근데 union find 로 푸는 게 더 나아 보이긴 하네여

## [최대 힙](https://www.acmicpc.net/problem/11279)

### 💡 아이디어 / 접근법

1. 최대힙 구현(개빡셈)
2. 0들어오면 pop
3. 아니면 push

### 🤔 후기

사실 이 문제의 난이도를 따지면 그냥 다른 언어였으면 실버 문제가 맞는데 JS 라면 골드 상위권까지 간다고 생각함. 왜냐하면 JS는 힙 라이브러리가 없기 때문에 class 로 싹 다 구현 해야함;; 처음에는 어찌저찌 풀어볼라다가 도저히 안되겠어서 구글링해서 클래스 어떻게 작성하는 지 부터 공부해야겠다 싶어서 참고를 많이했음; 그냥 힙 구현만 하면 날먹 문제;;
