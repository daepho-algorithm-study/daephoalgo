# 🧩 문제

## [궁금한 민호](https://www.acmicpc.net/problem/1507)

### 💡 아이디어 / 접근법

1. 인접 노드의 최단 거리가 적힌 입력값을 받음
2. 필요한 간선만 남기기 위해 2차원 배열을 하나 만듦
3. bfs 탐색으로 다음 값 뽑고 결과 배열에 저장
4. dist 배열에 대해 플로이드 워셜을 돌리면서 갈 수 있는 노드에 대해 최소값 찾기
5. 만약 dist[i][j] > dist[i][k] + dist[k][j] 의 경우라면 최단 거리라는 논리에 어긋나므로 바로 -1 출력
6. 만약 dist[i][j] === dist[i][k] + dist[k][j] 라면 dist[i][j] 간선 자체가 불필요하므로 Infinity 로 삭제 (반대가 안되는 경우는 i -> j 와 i -> k -> j가 같다고 해서 i -> k -> j 에 해당하는 경로를 삭제해 버리면 다른 경로들의 최단 거리 구성에 영향을 주기 때문 )
7. visited 체크를 위한 배열 생성 후 순회
8. 만약 삭제된 간선이거나 자기 자신이거나 이미 더한 간선일 경우 무시
9. answer 에 누적
10. 간선에 체크 표시(무향 그래프이기 때문에 반대 방향도 체크)

### 🤔 후기

플로이드 워셜에서 어떻게 하면 최소값을 유지하면서 모든 도로의 합을 구하는 경우를 구할 수 있을 지 생각을 진짜 많이 한 것 같음 그리고 가장 핵심인 간선 제거 부분에서 왜 역은 성립이 안되는 지 정말 헷갈리고 고민을 많이 했던 것 같음...

## [백양로 브레이크](https://www.acmicpc.net/problem/11562)

### 💡 아이디어 / 접근법

1. 자기 자신은 0으로 초기화
2. 일방 통행일 경우 반대 방향을 뚫어줘야 하므로 반대 방향에 비용 1 설정 아니라면 정,반대 방향 모두 비용 0 으로 초기화
3. 그 상태에서 플로이드 워셜을 돌리면 각 노드에 가기 위한 최소 비용이 저장됨 즉 최소 몇개의 도로를 양방향 통행으로 바꿔야 하는 지 저장됨
4. 그 후 입력 값에 해당하는 s, e 를 dist[s][e] 의 형태로 출력하면 됨

### 🤔 후기
 
문제는 쉬움 기본적인 플로이드 워셜 문제라고 생각을 함 근데 배열에 저장되는 최소 비용의 의미를 좀 생각해봐야 했음

## [우주 탐사선](https://www.acmicpc.net/problem/17182)

### 💡 아이디어 / 접근법

1. 입력으로 비용 행렬 받기
2. 모든 노드 간 최소 비용 계산 이유는 단순히 직접 이동 비용이 아니라 중간 노드를 거쳐 더 짧게 이동할 수 있는 경우까지 고려해야함 왜? 모든 노드에 대해 최소 시간을 찾으라고 했으니까
3. 따라서 플로이드 워셜을 통해 arr[i][j] 를 i -> j 로 가는 최소 비용으로 갱신함
4. 그 후 DFS(백트래킹) 을 이용하여 모든 경로를 탐색함
5. 시작 노드에서 모든 노드를 정확히 한 번씩 방문하는 경로를 탐색함
6. 만약 비용의 합이 이미 구한 최소값 보다 크거나 같다면 바로 종료
7. 모든 노드를 방문했으면 현재 경로의 비용으로 answer 갱신

### 🤔 후기
 
우선 계속해서 플로이드 워셜 문제를 봐서 그런가 크게 어렵다는 느낌은 안 받았음 입력값 그대로 플로이드 워셜 돌려서 모든 노드에 대해 최단 경로를 구하는 것 까지 쉽게 했음 근데 그 다음에 탐색을 어떻게 해야 K 에서 모든 노드를 정확히 한 번씩 탐색할 수 있을 지가 고민이었는데 외판원 문제와 동일하다고 생각되어 백트래킹으로 해당 문제를 해결했음 근데 백트래킹이 더 어려웠던 건 안 비밀..ㅎ
