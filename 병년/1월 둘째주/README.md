# 🧩 문제

**1.** [하노이탑 이동 순서](https://www.acmicpc.net/problem/11729)

### 💡 아이디어 / 접근법

1. 하노이탑의 경우 가장 큰 원판을 목표할 위치에 먼저 놓고 다음 원판을 치우는 방식
2. 따라서 먼저, N - 1 의 원판(N을 제외한 나머지 원판)을 A -> B 로 보내는 재귀 함수 호출
3. 그 다음 A -> C 로 N을 바로 보냄
4. 나머지 N - 1의 원판(N을 제외한 나머지 원판)을 B -> C로 옮기는 재귀 함수 호출

### 🤔 후기
하노이탑의 원리를 잘 알면 쉬운 문제였다고 생각했는데 그게 아니라면 좀 걸릴 거 같음 본인이 그랫음 ㅋ



**2.** [3 x n 타일링](https://school.programmers.co.kr/learn/courses/30/lessons/43164)

### 💡 아이디어 / 접근법

1. dp 배열 생성
2. 홀수일 경우 dp 배열에 저장 x
3. 짝수일 경우 그 이전의 prev(n - 2)의 제곱과 현재 값의 2로 나눈 값을 더해줌
4. 계속 n -= 2로 줄여줌
5. 결과 출력

### 🤔 후기
점화식을 세울 때 제곱과 현재 값의 2로 나눈 값을 더해주는 거는 금방 구했는데 완벽한 점화식이 아니라서
자꾸 실패해서 AI의 도움을 받고 풀었슴다... 그러니까 n 의 값도 계속 2씩 빼줘야 하드라구요...
dp 넘 어렵넹..

**3.** [혼자서 하는 틱택토](https://school.programmers.co.kr/learn/courses/30/lessons/160585)

### 💡 아이디어 / 접근법

1. o,x 카운트 세기
2. 가로, 세로, 대각선에 ooo 혹은 xxx 있는 지 체크해서 oWin, xWin 할당
3. 둘 다 개수가 0일 경우 1 리턴
4. 등등 틱택토에 필요한 로직을 조건으로 계속 걸어줌
5. 결과 출력

### 🤔 후기
처음 볼 때 문제는 너무 어려워 보였고 과거에 싸피에서 푼 틱택토가 생각나서 그런 문제인 지 알았는데 조건 검증만 하면 되는 문제였음
