### [오민식의 고민](https://www.acmicpc.net/problem/1219)

### 💡 아이디어 / 접근법
먼저 생각난 게 오민식은 비용을 무한하게 벌 수 있는 경우가 존재한다고 나와있음 이는 내야할 비용보다 받는 비용이 더 크며 결국 음의 사이클을 의미하므로 벨만 포드로 풀어야지 였고 또한 도착 노드를 갈 수 있는 지를 판별해야 했으므로 벨만 포드를 통해 음의 사이클일 경우 queue에 삽입 및 방문 체크 후 해당 queue를 이용하여 bfs를 수행하기로 하였음

1. N - 1 간선을 완화
2. edge의 개수 만큼 다시 완화 시도
3. 해당 시도에서 걸리면 음의 사이클이므로 queue에 다음 노드 삽입 및 방문 체크
4. 음의 사이클에 속하는 노드들에 대하여 bfs 수행
5. 만약 음의 사이클 내부의 노드가 end 라면 바로 true 반환
6. edge의 개수 만큼 반복을 돌려서 노드 값이 현재 음의 사이클 내부에 포함되어 있는 지 판별
7. 그렇다면 queue에 다음 노드 넣고 방문 처리 후
8. 탐색을 마치면 false 반환
9. true 라면 -1 출력
10. 만약 모든 탐색이 끝난 이후에 도착 지점이 갱신이 안됐으면 gg 출력
11. 도착지점이 갱신은 됐으나 flag가 true 라면 gee 출력 아니라면 해당 거리 출력


### 🤔 후기
이 문제는 벨만 포드 + BFS를 결합한 문제였는데 문제 자체가 굉장히 재밌었다고 생각이 되었음 내가 생각한 로직이 맞는 지도 헷갈렸는데 다행히 올바른 방향으로 접근했고 정답이 잘 나오는 걸 확인했음 근데 node 기준으로 자꾸 44%에서 실패하길래 로직이 잘못됐나 싶어서 python 으로 똑같이 코드를 구현하여 제출했는데 통과됐음... 아직까지 뭐가 문제인 지는 잘 모르겠는데 아마 Infinity 값 때문이라고 짐작은 하고 있음..ㅠ

### [골목길](https://www.acmicpc.net/problem/1738)
이 문제도 오민식이와 동일한 방향으로 접근을 했음 왜? 비용이 추가, 감소가 있기 때문에 이 문제는 음의 사이클이 발생할 수 있다고 판단을 해서 벨만 포드를 사용해야 한다고 생각했고 도착 지점이 명확하게 나와있고 해당 도착 지점에 갈 수 있는 지 판별하는 게 마지막 목표였기 때문에 bfs를 사용해야겠다고 생각했음. 근데 다른 점은 경로를 복원했어야 하는 문제였음

### 💡 아이디어 / 접근법
1. N - 1 간선을 완화
2. edge의 개수 만큼 다시 완화 시도
3. 해당 시도에서 걸리면 음의 사이클이므로 queue에 다음 노드 삽입 및 방문 체크
4. 음의 사이클에 속하는 노드들에 대하여 bfs 수행
5. 만약 음의 사이클 내부의 노드가 end 라면 바로 true 반환
6. edge의 개수 만큼 반복을 돌려서 노드 값이 현재 음의 사이클 내부에 포함되어 있는 지 판별
7. 그렇다면 queue에 다음 노드 넣고 방문 처리 후 부모 배열에 다음 좌표의 부모로 현재 좌표를 삽입
8. 탐색을 마치면 false 반환
9. true 라면 -1 출력
10. 만약 모든 탐색이 끝난 이후에 도착 지점이 갱신이 안됐으면 -1 출력
11. 도착지점이 갱신은 됐으나 flag가 true 라면 -1 출력 아니라면 while 문을 활용해서 path 배열에 현재 노드를 도착점(N)으로 설정하고 노드 값을 부모 배열을 계속 타고 올라가서 삽입하여 역추적 후 reverse() 하여 출력

### 🤔 후기
오민식이를 풀고나서 바로 풀었는데 금방 풀었고 실제로 로직은 완벽하게 동일했으나 경로 역추적이 추가가 되어있던 문제였음 근데 어차피 우리는 현재 노드에서 다음에 갈 노드의 정보를 쉽게 알 수 있기 때문에 음의 사이클 생성 부분에 parent 배열의 다음 좌표 값에 현재 노드를 삽입해 부모로 설정할 수 있었음 그 후는 로직이 동일하고 도착 지점에 방문할 수 있는 지 먼저 판별하고 해당 도착 지점이 음의 사이클 내부에 존재하지 않을 경우에 역추적 기법 즉 부모 배열을 타고 올라가도록 출력하여 문제를 해결했음.. 오민식도 그렇고 이것도 그렇게 내 인생 첫 플레 문제였는데 내가 생각한 방식이 맞았고 그렇게 결과가 나오는 것이 좀 뿌듯했음..ㅎ
