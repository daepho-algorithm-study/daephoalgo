# 🧩 문제

## [게임](https://www.acmicpc.net/problem/1072)

### 💡 아이디어 / 접근법

1. 승률 계산 후 99 이상이라면 바로 -1 리턴
2. 승률 계산 시에 판수(mid)를 X 와 Y 에 더해주고 계산
3. 그 후 승률이 변한 경우 right 값을 줄여서 더 탐색해보고 (판수가 적으면 더 좋기 때문에) 결과 값에 판수를 할당해줌

### 🤔 후기

어렵지 않은 문제였는데 새벽에 풀다보니 휴먼이슈가 발생해서 이해하는 데 조금 많이 걸렸다 판수를 어떻게 승률에 넣어줄 지 생각만 하면 쉬운문제

## [암기왕](https://www.acmicpc.net/problem/2776)

### 💡 아이디어 / 접근법

1. 수첩2의 원소만큼 반복문을 돌려서 수첩 1에 존재 하는 지 판별
2. 존재하면 result 배열에 할당 후 종료
3. 더 큰경우에는 left 값 늘려주고 아니라면 right 값 줄여주기

### 🤔 후기

두 가지의 배열이 주어지고 해당 배열을 통해 이분탐색을 진행하는 좋은 예시라고 생각했음

## [먹을 것인가 먹힐 것인가](https://www.acmicpc.net/problem/7795)

### 💡 아이디어 / 접근법

1. A 의 원소만큼 반복 돌리기
2. B[mid]의 값이 A 의 원소보다 작은 경우 left 값 늘리기 아니라면 right 값 줄이기
3. count 에 left 값 누적

### 🤔 후기

암기왕 문제랑 비슷해서 금방 풀었음

## [세 용액](https://www.acmicpc.net/problem/2473)

### 💡 아이디어 / 접근법

1. 세 가지의 용액을 뽑기
2. 세 가지의 용액을 뽑을 때 start, end 의 범위를 설정해주고 arr[i] + arr[start] + arr[end] 의 값을 구하고 최소값을 갱신해줌
3. 나머지는 두 용액과 같이 풀이해줌

### 🤔 후기

두 용액 문제와 동일한 문제였고, 차이점이라고 할 정도는 세 개를 뽑아야 하는 것 이었고 세 개를 뽑을 때 반복문 하나를 추가해서 해당점을 시작점으로 두고 그 사이의 값들의 합을 해주면 되는 문제였음

## [과자 나눠주기](https://www.acmicpc.net/problem/16401)

### 💡 아이디어 / 접근법

1. 과자들을 현재 기준으로 잡은 값(mid)로 나눔
2. 그 값들이 M 보다 크거나 같다면 left 값 늘리고 정답 갱신
3. 아니라면 right 값 줄이기

### 🤔 후기

과자를 나눈다는 의미가 좀 명확하게 와닿지 않아서 생각을 좀 했는데 어려운 문제는 아니었음
