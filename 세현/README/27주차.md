# 🧩 회고 문제

## [랜선 자르기](https://www.acmicpc.net/problem/1654)

### 💡 아이디어 / 접근법

- 이분 탐색 문제
- 랜선 중 가장 큰 걸 기준으로 left, right 잡기
- 위 랜선을 반으로 나눈 걸로 현재것 포함 다른 랜선들 모두 몫연산
- 모두 더해서 목표량(N)보다 적으면 right를 middle -1로, 목표량 이상이면 left를 middle + 1로 설정
- left가 right보다 작을때까지 계속 반복해주면 최대 랜선의 길이가 갱신됨

### 🤔 후기

- 처음에 left, right를 middle로 박아서 무한루프가 돔. 이분탐색 이해도가 부족했음
- 초기 left 0으로 두고 풀었었는데 이것도 틀림. 이분 탐색은 정답 후보를 left에서 right 사이로 찾는건데 길이 0짜리 랜선은 말이 안되는 후보였음
- 이분 탐색 공부를 더 해야겠다고 생각함

## [숨바꼭질](https://www.acmicpc.net/problem/1697)

### 💡 아이디어 / 접근법

- 1차원 bfs
- q에 가능한 위치 후보군을 계속 업데이트 해줌(위치와 시간을 동시에 기록)
- 찾는 위치가 나오면 바로 기록된 시간 반환

### 🤔 후기

- 처음에 까먹고 visited 안썼다가 메모리 터짐
- bfs 복습도 해놔야겠다고 생각이 듬
