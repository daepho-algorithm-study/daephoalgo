# 🧩 회고 문제

## [1로 만들기](https://www.acmicpc.net/problem/1463)

### 💡 아이디어 / 접근법

- 1차원 DP 선언
- 2부터 이전 연산 결과의 최솟값을 배열에 저장함
- 경우의 수에 따라 분기처리하여 가능한 연산만 반영하는 것

### 🤔 후기

- 처음에 연산 3가지 결과 모두 저장하려고 2차원 배열 생성함
- 이후 진행이 막혀버려서 힌트를 봄
- 1차원 배열로 처리한다고 해서 1차원 선언하고 곰곰히 생각해봄
- for 문 돌리면 차례대로 돌려야 하는데 그러면 N이 아니라 2부터 시작하면 되지 않을까 하는 아이디어가 떠오름 (사실 N부터 시작해도 됨)

## [쉬운 계단 수](https://www.acmicpc.net/problem/10844)

### 💡 아이디어 / 접근법

- 각 자릿 수는 맨 마지막 자리에 어떤 수가 오느냐에 따라 달라지는 것이 포인트
- 끝자리가 0, 9이면 무조건 1, 8이 올 수 밖에 없고 나머지는 앞뒤로 2개씩 올 수 있음
- 각 자릿수의 개수를 배열에다 저장해서 그 다음 자릿수의 개수 셀 때 꺼내 쓰기 

### 🤔 후기

- 이번엔 1차원 배열만 선언했었음(자릿수만 봄)
- 각 자릿수 0~9 까지의 경우를 모두 고려해야 돼서 2차원 배열로 선언해야 함

## [포도주 시식](https://www.acmicpc.net/problem/2156)

### 💡 아이디어 / 접근법

- 연속 3번이 아닌 경우를 정의
    - 3번을 빼기 (1 + 2)
    - 2번을 빼기 (1 + 3)
    - 1번을 빼기 (2 + 3)
- 3개를 놨을 때 가장 작은 인덱스를 dp에서 가져와야 함 (그 떄까지의 최대값을 가져오는 것)

### 🤔 후기

- 몇 번째를 dp에서 꺼내쓰는지 헷갈렸었음

## [가장 긴 증가하는 부분 수열](https://www.acmicpc.net/problem/11053)

### 💡 아이디어 / 접근법

- 실제 부분 수열을 만드는게 아니고 각 길이별로 가장 유리한 끝 값만 추가하여 길이를 찾음
- 예시로 A = [3, 8, 9, 1, 2] 가 있으면 과정이 다음과 같음
    - 3 -> [ 3 ]
    - 8 -> [3, 8]
    - 9 -> [3, 8, 9]
    - 1 -> [1, 8, 9]
    - 2 -> [1, 2, 9]
- 만약 이 뒤에 10 이라는 요소가 더 있으면 이다음 과정에서 LIS는 [1, 2, 9, 10] 이 되고
- 실제 최장 증가 수열은 [3, 8, 9, 10]이 되므로 길이는 둘 다 4임
- 즉 길이를 최대로 유지할 수 있음

### 🤔 후기

- 저번 스터디에서 설명 들었는데도 다시 보니까 이해가 잘 안됐음
- 부분 수열을 구현하는게 아니라는 점이 많이 헷갈렸던 것 같
- 수열의 관점이 아니라 길이의 관점으로 봐야되는 것을
