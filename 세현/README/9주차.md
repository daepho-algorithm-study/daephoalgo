# 🧩 문제

## [블랙잭](https://www.acmicpc.net/problem/2798)

### 💡 아이디어 / 접근법

- N 개의 카드 중 3개를 골라서 합을 M에 가깝게 맞추기 (nC3)
- 3중 포문으로 모든 경우의 수를 검사함

### 🤔 후기

- 처음에 비트연산으로 조합식 짜려고 했다가 실패함
- 비트연산은 nCr 할 때 n이 크면 시간이 기하급수적으로 늘어남
- nCr에서 n이 20 이하면서 r이 가변적일 때 쓰는게 효율적임
- r이 작게 고정되어 있고 n이 큰 경우에는 그냥 for문 돌리는게 차라리 낫더라

### 🤔 후기

- 이걸로 gpt한테 물어보니 각 자릿수는 제일 커 봐야 9이기 때문에 시작을 N-(9*자릿수 수) 부터 시작해줘도 된다고 함
- 바로 채용

## [체스판 다시 칠하기](https://www.acmicpc.net/problem/1018)

### 💡 아이디어 / 접근법

- 체스판을 잘라내서 가장 적게 고치는 횟수를 찾음
- 체스 패턴은 (가로 인덱스 + 세로 인덱스) % 2 로 표현될 수 있음 (체크무늬 패턴)
- 잘라낸 체스판은 첫 시작이 W인지 B인지 두 가지 경우의 패턴 모두 검사해야 함

### 🤔 후기

- 처음에 잘라낸 체스판 맨 처음 스타트 지점의 색깔에 따른 패턴만 검사함
- 그렇게 하면 고쳐야 하는 최솟값을 못 찾아냄
    - 반례
        ```
        BBWBWBWB
        BWBWBWBW
        WBWBWBWB
        BWBWBWBW
        WBWBWBWB
        BWBWBWBW
        WBWBWBWB
        BWBWBWBW
        ```


        - 첫 번째 칸이 `B`이지만, `W`로 시작하는 패턴이 더 적게 칠해질 수 있음
        - **B 패턴 (B로 시작)**: 1개 수정 필요
        - **W 패턴 (W로 시작)**: 33개 수정 필요
        - **정답**: 1
- 2가지 패턴 모두 생각하는 곳에서 진땀 뺌
