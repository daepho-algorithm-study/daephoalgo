# 🧩 회고 문제

## [노란불 신호등](https://school.programmers.co.kr/learn/courses/30/lessons/468371)

### 💡 아이디어 / 접근법

- 신호등마다 주기(사이클)와 노란불이 켜지는 시간, 끝나는 시간을 체크해 놓음
- +1초마다 모든 신호등이 노란불인지 다 검사함. 하나라도 없다면 그 시간은 정답이 아님
- 범위는 모든 사이클들의 최소공배수까지. 그 이상은 검사해도 의미가 없음
- 검사할 때 해당 시간을 사이클로 mod 해주면 항상 고정된 start - end 범위 내에서 검사할 수 있음

### 🤔 후기

- 처음에 set을 쓰려고 했는데 신호등이 최대 5개라 잘못된 접근법인걸 깨닫고 폐기
- 이차배열에다가 true, false 값 넣어서 일일이 다 비교해보려고 했는데 이것도 아닌거 같아서 폐기
- 시간을 아예 1초부터 끝까지 다 검사해 보는 방법을 떠올렸는데 범위를 특정하지 못해서 막혔음
- 결국 이 부분은 찾아봤는데 모든 주기의 최소공배수까지만 검사하면 된다고 함
- 구현 문제도 꾸준히 계속 풀어봐야겠다는 생각이 듬.

## [택배 상자 꺼내기](https://school.programmers.co.kr/learn/courses/30/lessons/389478)

### 💡 아이디어 / 접근법

- 상자가 지그재그로 쌓임
- 층(몫)과 그 위의 자투리(나머지) 를 잘 계산해야 함
- 방향도 잘 고려해서 예외처리 해줘야 함

### 🤔 후기

- 예외처리 생각하느라 꽤 고전했던 문제
- 3줄로 처리한 얼탱없는 다른사람의 코드가 인상깊었음(이해하기 힘들거같아서 해석하진 않음)
```
def solution(n, w, num):
    m1 = num%(w*2)
    m2 = ((w*2+1) - m1)%(w*2)
    # num 이상 n 이하의 수들 중 2*w로 나눈 나머지가 m1,m2인 것들의 수를 세면 된다.
    return len(range(num,n+1,w*2)) + len(range(num + (m2-m1)%(w*2), n+1, w*2))
```
