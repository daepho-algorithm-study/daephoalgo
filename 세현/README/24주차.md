# 🧩 회고 문제

## [인간-컴퓨터 상호작용](https://www.acmicpc.net/problem/16139)

### 💡 아이디어 / 접근법

- 누적합 배열을 2차원 배열로 만듬
  - 문자열의 문자 개수 길이의 방에다가 길이 26의 배열을 생성 (알파벳 26자)
  - 즉 각 문자열 자리에 어떤 문자가 몇번 왔는지를 기록하는 것임
  - 소문자의 아스키코드 범위가 97 ~ 122인 것을 이용함
  - 인덱스에다 + 97을 하면 a-z 의 소문자 아스키코드 값이 됨
- l 부터 r까지의 문자열 중 어떤 문자가 몇번 나왔는지 바로 검색

### 🤔 후기

- 누적합인데 기록을 26개 문자 전부 하는 게 특징
- 자바스크립트로 알고리즘 푸는 게 화가 났던 문제
- 백준 서브테스크 문제로 특이하게 50점, 100점을 맞을 수 있는데 js 코드는 자꾸 50점이 나옴
- 똑같은 코드를 C++로 바꿔서 제출하니 100점이 나옴 /ㅇoㅇ\;;
- 백준이 유독 js에만 기준이 빡빡하다고 들었는데 이 문제 풀면서 체감하게 됨

## [나머지 합](https://www.acmicpc.net/problem/10986)

### 💡 아이디어 / 접근법

- 누적합과 모듈러의 성질을 이용하는 문제
- 어떤 두 수 A-B를 모듈러 M 연산 했을 때 나머지가 같으면
  - _A - B_ 는 **M의 배수** 가 됨
  - A % M = B % M ----> A ≡ B (mod M)
- 누적합을 계산하는 방법은 가장 끝 요소 - 시작 요소임(가장 끝 요소가 배열의 맨 첫 부분부터 시작 요소 까지의 합을 포함하고 있기 때문에 빼줘야 함)
- 이를 A, B에 대입해보면 됨
- 즉, 누적합 배열에서 가장 끝 요소의 나머지와 시작 요소의 나머지가 같다면?
  - A - B 하면 M의 배수가 나옴
  - 즉, 나머지가 0임! (나누려는 M의 배수니까)
  - 따라서 각 누적합의 요소에서 나머지가 같은 것들 끼리 모아서
  - 2개 씩 (끝점, 시작점) 골라주면 됨
  - nC2 조합과 동일함

### 🤔 후기

- 정수론을 알고 있어야 풀리는 문제였음
- 수학 공부가 필요하다고 느꼈던 문제
- 저번에 공부했던 거 같은데 까먹은거 같은 느낌도 듬
- 반복적으로 숙달하는게 유일한 방법

## [체스판 다시 칠하기 2](https://www.acmicpc.net/problem/25682)

### 💡 아이디어 / 접근법

- 체스판이 격자로 다른 색이 칠해져야 함
- 두 가지 경우를 모두 고려함
  - 맨 처음(0,0)을 하얀색으로 칠하는 경우(W)
  - 맨 처음(0,0)을 검은색으로 칠하는 경우(B)
- 각 경우에서 색을 칠해야 하는 경우의 수를 카운트하는 누적합 배열을 만듬(누적합 배열 2개 만듦)
- 체스칸은 i행과 j열 인덱스를 합쳤을 때 홀수인 곳은 모두 같은 색이어야 하고 짝수인 경우도 모두 같은 색이어야 함
- 2차원 누적합 배열을 만드는 공식
  - prefix[i][j] = prefix[i-1][j] + prefix[i][j-1] - prefix[i-1][j-1] + arr[i][j]
  - 누적합 배열에서 위쪽 누적합 + 왼쪽 누적합 해주면 왼쪽 위 대각선의 누적합을 중복으로 빼주기 때문에 한번 더해줌
  - 그리고 원본 배열 해당 위치에서 로직을 통한 값을 더해주면 됨
- 만들어진 누적합 배열을 모두 슬라이딩(~K)으로 탐색해보면서 최솟값 찾기

### 🤔 후기

- 입력값을 잘못 파싱해서 몇시간 날렸던 문제
- ['B', 'W', 'B', ...] 이런 형태를 기대했는데 실제로 ['BWB...'] 이런 식으로 나옴
- 결과를 알고 많이 허탈했던 문제
- 좌표 계산하는 데 눈알이 빠지는 줄 알았음

## [회의실 배정](https://www.acmicpc.net/problem/1931)

### 💡 아이디어 / 접근법

- 대표적인 greedy 문제
- 회의가 끝나는 시간을 기준으로 정렬
  - 끝나는 시간이 같다면 시작하는 시간을 기준으로 정렬
  - 이를 안해주면 시작하는 시간과 끝나는 시간이 같은 회의를 처리하지 못함
- 순차적으로 탐색하면서 회의가 끝나는 시간 이후에(끝나는 시각 포함) 들어오는 회의 개수를 세어주면 됨

### 🤔 후기

- 처음에 시작하는 시간과 끝나는 시간이 같은 경우의 반례를 찾지 못해서 틀림
- 반례 찾는데 시간이 좀 걸렸던 것 같음
- tc는 다 통과해서 이 문제 코테에 나왔으면 틀렸을 거 같음..
