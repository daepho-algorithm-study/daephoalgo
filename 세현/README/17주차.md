## [N-Queen](https://www.acmicpc.net/problem/9663)

### 💡 아이디어 / 접근법

- 유명한 백트래킹 문제
- 비트마스킹으로 구현
- 비트마스킹 : visited를 배열이 아닌 정수로 구현하는 것. 속도가 상당히 빠름

### 🤔 후기

- 원래는 visited 배열 방식으로 한번 풀었다가 N-Queen이 비트마스킹으로도 유명한 문제여서 다르게 풀어봄
- 비트마스킹 이해하는데 시간이 꽤 걸림. (비트마스킹은 봐도봐도 새로움)
- 1 << x 를 원하는 핀포인트를 딱 집는다고 생각하니까 조금 편해짐
- 그리고 백트래킹할 때 visited를 바꿔주면 재귀부분 다음에 복구를 해줘야 하는데 (visited는 외부에 있어서 변경해줘야 바뀐 값으로 접근이 가능) 비트마스킹은 트릭이 가능해서 신박했음
    - 파라미터에다가 visited 값(Mask)을 할당하지 않고 식으로 바로 넘겨줘버리면 이전의 visited 값은 변경되지 않아서 복구할 필요 없음

## [스도쿠](https://www.acmicpc.net/problem/2580)

### 💡 아이디어 / 접근법

- 비트마스킹으로 백트래킹 구현
- N-Queen과 비슷하게 내가 썼던 숫자들을 다 체크해가면서 풀면 됨
- 가로줄 visited 9개, 세로줄 visited 9개, 박스(3x3) visited 9개를 관리해줘야 함
- box 번호는 (가로행 좌표를 3으로 나눈 몫) x 3 + (세로열 좌표를 3으로 나눈 몫) 해주면 해당 box 인덱가 나옴

### 🤔 후기

- 박스 인덱스 구하는게 어려워서 해메다가 인터넷 찾아봄(https://fbtmdwhd33.tistory.com/41)
- N-Queen에서 비트마스킹으로 구현했기 때문에 감을 더 잡고 싶어서 이 문제도 비트마스킹으로 구현해 봤음
- 비트마스킹은 다룰 수 있는 데이터의 개수가 적을 때 효과적임을 명심해야 할듯 (32 이내)
