# 🧩 문제

## [풍선 터뜨리기 - 2346](https://www.acmicpc.net/problem/2346)

### 💡 아이디어 / 접근법
deque을 사용하여 deque에 풍선 번호를 넣고, 배열에는 풍선 번호에 맞게 움직일 수 있는 숫자를 넣는다.
deque의 숫자를 뽑아내고 배열에서 해당 숫자에 해당하는 칸을 찾아서 풍선을 터트리는 걸 반복
움직이는 숫자가 양수면 오른쪽으로 이동하기 위해 앞에 숫자를 빼고 뒤에 넣음(이미 삭제 된 값이 있으므로 이동할 때 -1을 해주어야 한다.)
음수면 왼쪽으로 이동하기 위해 뒤의 숫자를 빼고 앞에 넣음


### 🤔 후기
덱을 사용하여 회전 시킬 때 음수와 양수를 나누어서 생각하여야 하는 문제였다. 양수 일 때는 이미 한칸 움직인걸로 생각해야 하기 때문에 움직이는 칸 수를 -1 해주어야 했는데 그걸 생각 못 해서 엉뚱한 값이 나왔었다.

# 🧩 문제

## [괄호 - 9012](https://www.acmicpc.net/problem/9012)

### 💡 아이디어 / 접근법
"("라면 stack에 넣고 ")"일 때 이전 값이 "("라면 짝이 맞으므로 스택에서 빼낸다. 
그렇게 빼냈을 때 스택이 비어 있다면 짝이 전부 맞으므로 YES를 출력 
아니라면 짝이 맞지 않으므로 NO를 출력 

### 🤔 후기
stack을 활용하여 괄호의 짝이 맞으면 값을 빼내는 식으로 풀었다. 하지만 "(" 만 넣고 ")"는 안넣다 보니 ")"가 먼저 들어왔을 때도 스택이 비어있어 값이 제대로 출력 되지 않아 조건을 만족시키지 못 할 때 ")"를 삽입했다.

# 🧩 문제

## [도키도키 간식드리미 - 12789](https://www.acmicpc.net/problem/12789)

### 💡 아이디어 / 접근법
현재 값을 두고 현재 값과 처음 입력 받는 값이 다르면 기다리는 줄인 stack에 삽입 
현재 값이랑 입력 받는 값이 같다면 다음 번호로 이동
그리고 기다리는 줄에서 해당 차례가 왔다면 꺼냄 이렇게 해서 스택이 비게 되면 간식을 먹을 수 있고 스택에 번호가 남아있다면 간식을 못 먹는다.

### 🤔 후기
처음에는 숫자를 먼저 출력해야 하기 때문에 queue로 풀어야 하는거 아닌가라고 생각했는데 굳이 그럴 필요가 없었다.
입력을 받으면서 입력 값이 현재 차례가 아니라면 대기줄인 stack에 다 담아주면 됐기 때문

# 🧩 문제

## [queuestack - 24511](https://www.acmicpc.net/problem/24511)

### 💡 아이디어 / 접근법
각 인덱스 별로 큐와 스택을 실행하게 된다. 예제를 한번 돌려보면
c배열의 2가 삽입 된다면
큐 => [1, 2] 1이 빠지고 [2]가 남게 되고 1이 빠진다
스택 => 1이 빠졌으므로 1이 삽입되어 [2, 1]이 되는데 스택은 후입선출이므로 다시 1이 빠져 [2]가 남는다.
스택 => 1이 빠져있으므로 [3, 1]이 되고 스택이므로 다시 1이 빠진다.
큐 => 1이 빠져있으므로 [4, 1]이 되고 큐는 선입선출이므로 4가 빠지고 [1]이 남게된다. 그래서 가장 나중에 빠지는 4를 먼저 출력

이렇게 규칙을 따라가다 보면 결국 큐에서만 값이 바뀌는걸 확인 할 수 있다. 그래서 자료구조가 큐인 0인 부분만 덱에 전부 삽입하고 가장 나중에 있는 큐가 가장 먼저 빠지므로 c배열의 값을 앞에 삽입하고 덱의 가장 뒷 부분을 빼서 출력한다.

### 🤔 후기
각 인덱스마다 큐와 스택을 적용하는거여서 문제를 이해하는데 엄청 오래걸렸다. 결국 큐에서만 바뀌는 문제라 이중 for문으로 인덱스를 바꾸듯이 접근했지만 시간초과로 덱을 사용
