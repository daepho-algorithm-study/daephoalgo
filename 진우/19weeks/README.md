# 🧩 문제

## [유기농 배추 - 1012](https://www.acmicpc.net/problem/1012)

### 💡 아이디어 / 접근법
X, Y 좌표를 입력 받으므로 좌표에 해당하는 배열 인덱스를 1로 만들어준다.
델타탐색과 dfs를 활용하여 해당 군집을 탐색해주고 dfs를 탈출하게 되면 해당 군집이 끝났다는 것이므로 cnt +1을 해준다.
이 문제는 여러개의 테스트케이스가 들어오므로 출력 이후 배추 배열과 방문 배열을 초기화 해줘야 한다.   

### 🤔 후기
N과 M, X와 Y를 반대로 사용하여 엉뚱한 값이 나왔었다. 행과 열에 대한 구분을 잘 해야한다.

# 🧩 문제

## [DFS와 BFS - 1260](https://www.acmicpc.net/problem/1260)

### 💡 아이디어 / 접근법
dfs와 bfs의 결과를 함께 출력해야 한다. dfs 함수를 호출하고 줄 바꿈을 한후 bfs 함수를 호출하고 줄 바꿈을 하였다.
여러개의 테스트케이스가 주어졌다면 방문 배열을 매번 초기화 했을테지만 이 문제에선 한 개의 테스트케이스를 가지고 dfs와 bfs를 하기 때문에 dfs의 방문 배열과 bfs의 방문 배열을 각각 생성해서 풀었다.
그리고 dfs와 bfs의 함수에서 각 노드를 방문할 때마다 노드 번호를 출력하면 된다.

### 🤔 후기
노드를 출력해야 하는데 기저조건을 사용해야 출력이 되는지 생각하다가 굳이 필요없다는 것을 알게 되었다.

# 🧩 문제

## [바이러스 - 2606](https://www.acmicpc.net/problem/2606)

### 💡 아이디어 / 접근법
1번 컴퓨터와 연결된 모든 노드들이 바이러스에 감염되므로 dfs를 통한 완전 탐색을 통해 해당 노드를 방문할 때마다 cnt에 +1을 해준다. 1은 제외하고 감연된 컴퓨터 수를 묻기 때문에 cnt = 0부터 시작

### 🤔 후기
dfs의 정석 문제였다.

# 🧩 문제

## [단지번호붙이기 - 2667](https://www.acmicpc.net/problem/2667)

### 💡 아이디어 / 접근법
입력이 공백없이 주어지기 때문에 각 줄을 문자열로 입력 받아 반복문을 통해 배열에 해당 인덱스의 값으로 바꿔준다.
그리고 인접한 상하좌우로 인접한 집들은 하나의 단지이기 때문에 델타탐색과 dfs를 함께 활용하여 완전 탐색을 진행한다.
dfs를 탈출하게 되면 하나의 단지가 완성된 것으로 보고 cnt의 값을 출력을 위한 배열에 삽입한다.
그리고 출력 배열의 크기를 먼저 출력하고 출력 배열의 인덱스를 통해 하나하나 출력하면 된다.


### 🤔 후기
델타 탬색 + dfs의 정석이었다. 오랜만에 델타 탐색을 사용하여 좀 해맸던것도 있고, 입력을 받는게 좀 머리 아팠다.
