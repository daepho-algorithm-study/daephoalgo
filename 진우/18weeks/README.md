# 🧩 문제

## [RGB거리 - 1149](https://www.acmicpc.net/problem/1149)

### 💡 아이디어 / 접근법
색상은 R, G, B 3개로 고정이고 각 N번째 집마다 색을 칠하기 때문에 두 번째 집부터 비용을 계산한다. 만약 R이라면 이전 집은 G, B 둘중 하나이고, 나머지도 동일하게 계산하여 값을 전부 더해준 다음 마지막 N번째 중에 최소 값을 출력하면 된다.

### 🤔 후기
R, G, B로 나눠서 더 해주면 다음 집을 선택했을 때 이전 집과 색이 겹칠 수도 있지 않을까?라고 생각했지만 그 이전에서 이미 동일한 색상이 아닌 값들을 누적하였으므로 겹칠일이 없었다.

# 🧩 문제

## [01타일 - 1904](https://www.acmicpc.net/problem/1904)

### 💡 아이디어 / 접근법
길이가 0일 때 경우의 수: 1
길이가 1일 때 경우의 수: 1
이렇게 되고 처음은 1 또는 00으로 시작하므로
처음이 1일 때 경우의 수: N - 1
처음이 00일 때 경우의 수: N - 2 가 되므로
피보나치 수열의 점화식을 따라가게 된다.
하지만 수가 커지만 정답이랑 달라질 수 있기 때문에 값을 문제에서 주어진 15746으로 미리 나누어 dp에 저장한다.

### 🤔 후기
피보나치 수열의 점화식을 따라간다는걸 발견했지만 15746으로 나눠주지 않아 틀렸었다.

# 🧩 문제

## [연속합 - 1912](https://www.acmicpc.net/problem/1912) 

### 💡 아이디어 / 접근법
연속된 수를 더해야 하고 그 연속된 합의 모든 경우 중에 큰 값을 출력해야 한다. 그래서 두 번째부터 시작하여 dp에 저장된 이전 값과 기존 배열에 저장된 현재 값을 더했을 때 배열에 저장된 현재 값 보다 크다면 그 값을 더해서 계속 연속되게 이어가고, 그게 아니라면 현재 값이 더 크기 때문에 현재 값으로 값을 바꾼 다음 이어가도록 한다. 그 다음 dp배열에서의 최댓값을 출력하면 된다.
 
### 🤔 후기
어떻게 연속된 값의 합의 모든 경우를 구할지 고민하다가, 결국 힌트를 보았다. 이전 연속된 값의 합이 현재 값 보다 작으면 어차피 그 값들은 사용하지 못 하는 값이기 때문에 그 현재 값부터 다시 계산하도록 해야 하는 문제였다.

# 🧩 문제

## [정수 삼각형 - 1932](https://www.acmicpc.net/problem/1932)

### 💡 아이디어 / 접근법
삼각형의 줄 수와 각 줄에는 삼각형의 줄의 번호만큼 숫자 갯수가 주어지기 때문에 i는 N까지 j는 i까지로 하여 2차원 배열로 입력 받고 나머지 부분을 0으로 채운다.
그리고 다음 입력 값은 이전 줄의 숫자이고 그 중에서 2개만 뽑을 수 있으므로 현재 j에 대해 [i - 1][j - 1], [i - 1][j]이 성립한다. 이전 값들 중 큰 값을 찾아 계속 더해주고 N번째의 값중에 최댓값을 출력하면 된다.

### 🤔 후기
이차원 배열로 입력 받아 입력 받은 값을 제외하고 전부 0으로 채우면 되겠다고 생각은 했지만, 입력을 어떻게 받아야 할지 몰라 힌트를 참고하였다. 힌트를 통해 j를 i까지만 하면 입력 받을 부분을 배열에 채울 수 있는걸 알게되었다.

## [파도반 수열 - 9461](https://www.acmicpc.net/problem/9461)

### 💡 아이디어 / 접근법
문제에 주어진 값을 따라가다 보면 피나보치 수열과 비슷한 형태를 보인다. 하지만 이 파도반수열은 1~3까지의 값이 전부 1이고 그 이후로 P(n) = P(n-2) + P(n-3)의 점화식을 따른다.

### 🤔 후기
배열을 0부터 시작하였다가 계산 값이 꼬여 0번을 0으로 두고 1부터 시작하도록 하였다. 점화식만 찾을 수 있다면 쉽게 풀 수 있는 문제
