# 🧩 문제

## [체스판 다시 칠하기 - 1018](https://www.acmicpc.net/problem/1018)

### 💡 아이디어 / 접근법
만약 w로 시작한다면 i,j 좌표값의 합이 짝수이고, b는 홀수가 된다. (b로 시작한다면 그 반대)
총 N, M의 범위에서 8, 8씩 끊어서 전부 확인하고 w와 b로 시작하는 것 중 작은 값을 출력


### 🤔 후기
i,j의 좌표값을 활용해서 짝수 홀수로 나누는 것을 생각하기가 어려웠음 
그리고 8칸씩 자를 때의 범위 설정도 어려웠음

# 🧩 문제

## [설탕 배달 - 2839](https://www.acmicpc.net/problem/2839)

### 💡 아이디어 / 접근법
첫 번째
딱 떨어지면서 가장 적은 봉지를 챙겨야 하기 때문에
while문을 N 돌면서 5의 배수이면 몫을 cnt에 더하고 아니라면 3을 빼고 cnt에 1을 더한다.
그렇게 3을 빼가면서 5의 배수인 값이 있을 때 출력, 없다면 while을 빠져 나온 후 -1 출력

두번째
N을 5로 나눈 몫으로 for문을 실행 최대값인 뒤에서부터 돌면서 N에서 5의 배수만큼 빼고 3으로 나누어 떨어지는지 확인
3으로 나누어 떨어진다면 3으로 나눈 몫을 더하고 개수 return 아니라면 -1 return


### 🤔 후기
문제는 쉬웠지만 막상 로직을 구현하려고 하니 생각보다 어려웠다. 
while문의 탈출 조건 설정과 탈출 후 출력 값을 어떻게 해야할지 고민이 많았는데
while에서 출력을 해버리면 쉽게 해결 가능했다.
