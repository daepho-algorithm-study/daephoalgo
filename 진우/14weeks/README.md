# 🧩 문제

## [다리 놓기 - 1010](https://www.acmicpc.net/problem/1010)

### 💡 아이디어 / 접근법
- 첫 번째
조합식의 각 항을 분리하여 누적으로 곱해 준다.
<details>

```
# M C N 계산 (팩토리얼 없이 곱셈과 나눗셈으로 직접 계산)
    for i in range(1, N + 1):
        result = result * (M - N + i) // i  # //는 정수 나눗셈

```
</details>

- 두 번째
dp를 활용하여 이전 경우의 수를 더 해준다.
=> 파스칼 항등식에 의해 다음이 가능하다
nCr = (n-1)C r + (n-1)C(r-1)

<details>

```
long long dp[31] = { 0 }; // dp[j]는 j개를 선택하는 조합의 수 저장 (최대 30까지)
        dp[0] = 1;  // 0개 선택(nC0) 경우의 수는 항상 1

        // M개의 항목을 순서대로 고려하며 조합 계산
        for (int i = 1; i <= M; i++) {
            // 뒤에서부터 업데이트하여 이전 단계의 값을 덮어쓰지 않도록 함
            for (int j = i; j >= 1; j--) {
                dp[j] = dp[j] + dp[j - 1]; 
                // dp[j] : i-1개 항목에서 j개 선택한 경우
                // dp[j-1] : i-1개 항목에서 j-1개 선택한 경우
                // i번째 항목을 포함하지 않는 경우 + 포함하는 경우
            }
        }

```
</details>

### 🤔 후기
그냥 팩토리얼을 구현하게 되면 정수 오버플로우가 발생해 엉뚱한 값이 출력되거나 출력되지 않는 경우가 발생한다. 그래서 조합식을 변형하거나 파스칼 항등식을 이용해 dp를 활용하는 방법을 사용했어야 했다.

# 🧩 문제

## [Average Speed - 4366](https://www.acmicpc.net/problem/4366)

### 💡 아이디어 / 접근법
한줄로 입력 받아 문자열을 파싱
8자리를 파싱하고 그 8자리 문자열에서 시간, 분, 초를 또 나누어 파싱한 다음 정수로 변환하여 시간을 저장
그리고 이동거리를 파악해야 하기 때문에 시간들을 초로 변환하여 현재 시간에서 이전 시간을 빼주고 속도를 곱한다.
속도 변화가 없는 구간에서 거리를 출력해야 하므로 전체 길이가 8이상인 경우 속도를 갱신해주고 아니라면 출력하도록 한다.

### 🤔 후기
처음에는 시간으로 전부 변환하여 계산하였는데 그러게 되니 소수 계산에 의해 엉뚱한 값이 출력되는 경우가 발생하였다. 그리고 속도를 먼저 갱신했을 때도 0이 아니라 현재 시간에 속도를 곱한 값이 발생하여 속도 갱신을 나중에 하도록 수정하였다.
