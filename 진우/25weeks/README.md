힙이 비어있으면 0 출력하고 최소 힙이므로 가장 작은 값이 top()되므로 top()을 출력하고 큐에서 제거 해준다.

### 🤔 후기

# 🧩 문제

## [N번째 큰 수 - 2075](https://www.acmicpc.net/problem/2075)

### 💡 아이디어 / 접근법
우선순위큐를 활용하여 최소힙으로 오름차순 정렬을 하게 되면 가장 작은 값이 top이 되므로 N번만큼 큐에 삽입하면 top이 N번째 값이 된다.
현재 들어갈 값이 top보다 크면 큐에서 제거하고 새로 넣는 식으로 반복하면서 N만큼의 크기를 유지한 후 top을 출력

### 🤔 후기
우선순위큐를 최소힙으로 하여 크기를 N만큼 유지하면 되는데 이걸 생각하기가 어려웠다.

# 🧩 문제

## [나무 자르기 - 2805](https://www.acmicpc.net/problem/2805) 

### 💡 아이디어 / 접근법
이분탐색을 활용하여 처음은 아무것도 자르지 않는 0, 마지막을 나무의 최대 높이로 설정하여 이분탐색을 해준다.
그렇게 각 배열의 나무에서 mid 값을 뺀 값을 더해주고 그 값이 가져가고자 하는 나무의 길이가 된다면 해당 mid 값이 최대의 길이이므로 출력해준다.
 
### 🤔 후기
아래서에부터 시작하기 때문에 정렬을 하지 않고도 이분 탐색을 활용할 수 있는 문제였다.

# 🧩 문제

## [두 수의 합 - 3273](https://www.acmicpc.net/problem/3273)

### 💡 아이디어 / 접근법
투포인터를 활용하여 첫번째 인덱스를 시작, 마지막 인덱스를 끝으로 두고 한 지점을 고정하여 현재 합이 목표 값이 되면 cnt를 증가시켜주고 합이 목표값 보다 작다면 마지막을 고정하고 시작점을 증가시키고 
반대로 목표값 보다 작다면 마지막을 감소 시키고 시작 값을 증가 시킨다.

### 🤔 후기
투포인터를 활용하면 쉽게 풀리는 문제였다. 하지만 투 포인터도 정렬을 해야 하는지 헷갈렸다.

# 🧩 문제

## [절댓값 힙 - 11286](https://www.acmicpc.net/problem/11286)

### 💡 아이디어 / 접근법
우선순위큐에 pair을 활용하여 절대값의 값과 현재 값을 쌍으로 넣어준다.
이후 큐의 정렬 기준을 절댓값 작은 수 우선으로 하도록 하게 되면 실제 값 작은 수(음수) 우선하게 정렬되므로
top의 실제 값을 출력해준다.

### 🤔 후기
우선순위 큐의 정렬을 활용하는 문제였는데 푸는게 쉽지 않아서 정답을 참고했다.
