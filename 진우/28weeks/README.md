# 🧩 문제

## [회의실 배정 - 1931](https://www.acmicpc.net/problem/1931)

### 💡 아이디어 / 접근법
최대한 많은 회의를 잡으려면 회의가 끝나는 시간이 빠른 순서대로 회의를 잡으면 된다.
그래서 종료 시간 순서대로 넣고 오름차순 정렬을 통해 빠른 순서를 파악한다.
현재 선택된 회의가 끝난 시각 보다. 조사 중인 회의의 시작 시간이 같거나 늦다면
회의 확정하고, 개수 증가시킨 후 기준 시각을 현재 회의가 끝나는 시간으로 갱신한다.

### 🤔 후기
그리디라는걸 몰랐으면 어떻게 풀어야 할지 막막했을거 같다. 종합적으로 봤을 땐 최적이라는 보장은 절대 없기 때문에 어떻게 회의 수를 가장 많이 가져갈지 계속 고민했을 것

# 🧩 문제

## [나머지 합 - 10986](https://www.acmicpc.net/problem/10986)

### 💡 아이디어 / 접근법
S[j] % M == S[i] % M 이면, 구간 [i+1, j]의 합은 M의 배수라는 원리 이용한다.
반복문을 통해 현재까지의 누적합 갱신하면서 누적합을 M으로 나눈 나머지 계산하고,
0번 인덱스부터 현재(i)까지의 합 자체가 M으로 나누어떨어지는 경우 갯수를 증가시킨다.
이전 지점들 중 누적합의 나머지가 현재와 같은 지점들을 저장해두고, 현재 지점의 나머지 횟수를 기록하여 다음 지점들이 참조할 수 있게 한다.
현재 저장된 지점과 같은 나머지를 가진 값이 나오면 그 지점들과 연결할 수 있기 때문

### 🤔 후기
S[j] % M == S[i] % M 이면, 구간 [i+1, j]의 합은 M의 배수라는 원리는 이해했지만, 구간을 배열에 저장하여 갯수를 센다는 생각을 하지 못 했다. S[j] % M == S[i] % M이면 어차피 각 구간간의 합이기 때문에 그 구간합을 시작점으로 현재 구간합을 끝점으로 생각하니 이해하기 편했다.

# 🧩 문제

## [체스판 다시 칠하기 2 - 25682](https://www.acmicpc.net/problem/25682)

### 💡 아이디어 / 접근법
이전에 활용했던 2차원에서의 누적합 공식을 이용한다.
첫 번째가 B일 경우와 W일 경우 두 가지 경우를 나누어 배열로 누적합을 저장하고, 현재 해당하는 위치가 바꿔야 하는 부분이면 현재 위치의 값을 증가시킨다.
그 이후 W 누적합과 B 누적합을 K * K 범위만큼 순회하면서 가장 작은 값을 찾아낸다.
 
### 🤔 후기
이전에 풀었던 이차원에서의 누적합 공식과 체스판 다시 칠하기1의 응용 문제. 이전 누적합은 원래 배열의 현재 인덱스 값을 더해줘야 하지만 여기선 원본 배열이 문자열이기 때문에 저장한 배열에서의 값과 다르면 현재 인덱스의 값을 증가시키는 것으로 누적합을 사용하였다.
