# 🧩 문제

## [통계학 - 2108](https://www.acmicpc.net/problem/2108)

### 💡 아이디어 / 접근법
입력 받는 숫자의 범위가 절대 값 4000을 넘지 않기 때문에 -4000~4000까지의 범위로 배열을 만들어 두고 최빈 값을 바로 저장 했다 입력 받는 숫자에 4000을 더하게 되면 0이 -4000이 되는식으로 최빈 값을 저장해 최빈값에 해당하는 값을 다시 불러올 때는 -4000을 하여 원래 값으로 만들고 배열에 넣어 정렬하여 1번째 인덱스를 출력

### 🤔 후기
평균이나 범위 중앙값을 구하는 것은 어렵지 않았지만 최빈값을 구하는게 생각보다 어려웠다. 처음에는 map을 통해서 key-value로 최빈값을 저장했으나 결국 필요한건 value가 아닌 key의 값이라 map의 key 값 자체를 불러오는게 힘들어 정적배열을 활용해 최빈값을 저장하였다.

# 🧩 문제

## [영단어 암기는 괴로워 - 20920](https://www.acmicpc.net/problem/20920)

### 💡 아이디어 / 접근법
key-value를 활용해 빈도수를 저장하고 key를 호출하여 빈도수가 많으면 정렬할 수 있도록 compare 함수의 조건문을 작성, 그리고 중복을 제거 해야 하기 때문에 일단 set을 활용해 중복을 제거하고 그 제거한 단어를 정렬하여 출력해야 하기 때문에 배열에 다시 담아서 정렬 조건에 맞게 정렬 되도록 한다.

### 🤔 후기
set을 vector로 바꾸는 방법을 몰라서 애를 먹었다. 그리고 compare 조건식만 잘 작성해주면 알맞게 출력이 된다.

# 🧩 문제

## [인사성 밝은 곰곰이 - 25192](https://www.acmicpc.net/problem/25192)

### 💡 아이디어 / 접근법
ENTER가 입력된 후 첫 번째 메세지만 인사이고 나머지는 인사가 아니기 때문에 set에 넣어 중복을 제거 해준다. 그리고 ENTER 이후의 나오는 사람들을 집합에 추가 해주고 다시 ENTER가 나온다면 집합의 크기를 더해주고 다시 숫자를 세기 위해 집합을 초기화 해준다. 그리고 반복문이 끝난다면 집합에 남아있는 사람들도 인사를 한 것이기 때문에 결과에 더해주고 출력

### 🤔 후기
처음에는 무조건 ENTER가 들어오기 때문에 첫 번째 ENTER를 제외하고 하는 방법도 있었겠지만, 그런 조건을 설정하는게 생각보다 머리 아파서 그냥 ENTER를 받을 때마다 값을 더해주고 초기화 해줬다. 처음 ENTER를 받았을 때 집합의 크기가 0이기 때문에 더해줘도 결과에 영향을 주지는 않기 때문에

### 💡 아이디어 / 접근법
