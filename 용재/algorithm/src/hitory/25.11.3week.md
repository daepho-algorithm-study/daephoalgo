# 🧩 후기 문제

## [숫자 카드](https://www.acmicpc.net/problem/10815)

### 💡 아이디어 / 접근법

- 처음엔 걍 들어오는거 전부 배열에 담아서 비교함
- 시간초과 날거 예상하고 구현했는데 바로 시간초과 발생
- 그래서 배열의 크기를 숫자의 범위로 만들고 들어오는 수에 + 10,000,000해서 해당 숫자 인덱스에 +1해줌
- m의 배열을 따로 만들어서 마지막에 비교할 숫자 저장
- 전체 배열을 순회하여 m과 비교후 출력

### 🤔 후기

- 500,000 * 500,000을 20,000,000으로 축소하니 시간초과 발생하지않고 풀이 성공
