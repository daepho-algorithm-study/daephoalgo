# 🧩 후기 문제

## [정보섬의 대중교통](https://www.acmicpc.net/problem/28113)

### 💡 아이디어 / 접근법

- 처음에는 while문을 통해 실제로 시간이 반복문 1회마다 1씩 증가하게 짜려고 했었음
- 근데 그러면 굳이 필요없는 시간을 소모한다고 생각해 O(1)로 풀 수 있게 if문으로 변경했음
- 일단 버스가 오는 시간 a가 지하철이 오는시간 b보다 빠르면 고민할 것 없이 버스를 태우고
- 그 반대의 경우와 같을 경우를 고려해야하는데
- 일단 버스가 만약 더 늦게 온다면 지하철 이동시간을 고려해야했음 만약 지하철이 10분뒤 도착하는데 내가 지하철 역 까지 가는데 11분이 걸린다면 총 21분을 소모하기 때문
- 해당 사항을 떠올려 수식을 찾아냈고 문제풀이 성공

### 🤔 후기

- 머리속으로는 경우마다 어떤게 더 빠를지 생각하는게 쉬웠지만 수식으로 풀어쓰려니 막상 막힌 문제
- 브론즈5 문제지만 사고력을 키우는데 도움을 받았음

## [Every Second Counts](https://www.acmicpc.net/problem/15080)

### 💡 아이디어 / 접근법

- 입력이 2번으로 정해져있으니 따로 for문으로 입력 받는게 아니라 걍 StringTokenizer 두번 써서 두줄 입력 받음
- 이후 이전시간, 이후시간을 받는데 콜론의 경우 쓸모없는거라 문자열로 받은 뒤 정수 변환 할까 했다가
- 콜론만 문자열로 받고 나머지는 인트로 파싱
- 이전, 이후 시간 전부 초로 환산한 뒤에 뺄셈 해주는데 음수값 나오면 하루 뒤라는 소리라 24*3600 더해주고 마무리

### 🤔 후기

- 처음에는 시가 음수면 +24해주고 분이 음수면 시에서 -1 분에 +60 이렇게 했다가 하루 넘어가는 경우를 고려하지 못해 틀림
- 이후에 풀이 방식 변경으로 풀었음
- 예전에 파이썬으로 비슷한 문제를 풀었던 기억이 있는데 그때도 이방법이었으면 엄청 쉽게 풀었을듯

## [공백 없는 A+B](https://www.acmicpc.net/problem/15873)

### 💡 아이디어 / 접근법

- 3자리가 넘어가는지 안넘어가는지 먼저 판별했음
    - 3자리가 넘어가면 a혹은 b 둘 중 하나는 무조건 10이고, 나머지 하나의 수만 찾으면 되기 때문(문제의 조건이 0 < A,B <= 10)
    - 4자리라면 둘 다 10임
    - 2자리의 경우엔 각각 할당해 주어 문제 해결

### 🤔 후기

- 처음에 반복문으로 숫자 할당 해주려다가 길이따라서 조건 바뀐다는것 깨닫고 방향 수정했음
- 3자리일 때 b가 10이되는 경우를 생각 못해서 한번 틀렸음..
- 자바에서 Integer.valueOf()를 사용해도 -'0'을 해주지 않으면 아스키코드 숫자가 나오는점을 간과해 생각보다 오래걸림
