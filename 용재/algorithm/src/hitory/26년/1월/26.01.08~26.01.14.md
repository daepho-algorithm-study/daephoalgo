# 🧩 후기 문제

## [이분 그래프](https://www.acmicpc.net/problem/1707)

### 💡 아이디어 / 접근법
- 각 노드를 탐색하면서 color에 해당하는 숫자를 visited에 남긴 뒤 정점을 이동한다.
- 이동을 하다 같은 색의 노드를 만난다 = 이분 그래프가 아니다 로 접근을 했고 그 방향이 맞았다.
- 또한 모든 그래프가 이어져있다는 보장이 없었기 때문에 모든 그래프를 탐색하여 문제 풀이를 진행했다.

### 🤔 후기
- 문제에서 각 집합에 속한 정점끼리는 인접하지 않도록 분할할 수 있을때 해당 그래프를 이분 그래프라 부른다. 라고 설명이 되어있어 이분 그래프를 이해하는것이 조금 힘들었다. 검색을 통해 이분 그래프를 더 빠르게 이해할 수 있었으며, 해당 검색이 없었다면 이분 그래프를 이해하지 못하고 못풀었을 문제였다.
- [이분그래프를 이해하는데 도움을 준 블로그](https://gmlwjd9405.github.io/2018/08/23/algorithm-bipartite-graph.html)

## [빙산](https://www.acmicpc.net/problem/2573)

### 💡 아이디어 / 접근법
- 먼저 빙산의 덩어리 수를 세기 위해 덩어리를 세는 함수, 그리고 해당 함수에서 사용할 bfs 함수
- 빙산을 녹이는 함수 총 3개의 함수를 사용해 문제를 풀었다.
- 빙산을 녹이는 함수를 사용하지 않고 한번에 처리하려고 하니 한개의 빙산이 과도하게 많이 녹는 문제가 발생해 해당 방식을 선택하였다.

### 🤔 후기
- 문제 자체는 직관적이었으나 해당 방식을 구현하려면 여러개의 함수가 생기고 해당 함수끼리 언제 동작해야하는지,
- 배열은 언제 초기화하는게 맞는지, 종료조건은 어떻게 잡는지에 대해 생각하는것이 조금 어려웠던 문제였다.

## [토마토](https://www.acmicpc.net/problem/7576)

### 💡 아이디어 / 접근법
- 직전에 빙산을 풀었기 때문에 토마토 익은걸 판별하는 함수, 이번 시간에 익을 토마토를 찾는 함수, 토마토를 익게하는 함수로 접근했었다.
- 해당 방식으로 풀이를 하니 최악의 경우 하루에 하나씩 n*m좌표를 전부 확인해야 하는 문제가 발생할 수 있어 시간초과가 발생했다.
- 이후 while문을 통해 시간을 추가하는것을 포기하고,
- bfs를 한번에 여러 위치에서 진행을 하면서 box 좌표에 현재시간+1을 해 시간을 직접 기록하는 방식을 통해 시간을 측정했다.
- 한번에 bfs가 가능한 모든 좌표에서 출발했기 때문에 반복문을 통해 0(안익은 토마토)이 남아있다면 익지 못하는 토마토가 있기 때문에 -1출력을 하면 됐다.

### 🤔 후기
- 방식은 머리에 떠올라 구현했으나 시간복잡도를 제대로 고려하지 못해 풀이에 실패했다.
- 여러 위치에서 bfs를 구현하는 문제가 처음이라 조금 더 어렵게 느껴졌으며 해당 유형에 대한 공부가 필요함을 느꼈다.
