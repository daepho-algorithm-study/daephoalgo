# 🧩 후기 문제

## [색종이 - 2](https://www.acmicpc.net/problem/2567)

### 💡 아이디어 / 접근법
- 처음에는 전체 도화지에 색종이를 칠한뒤 bfs를 사용하여 해결하려고 했다.
- 구현을 하다보니 실제 종이보다 과대평가하는 문제가 생겼는데 해당 문제를 해결하기위해 고민하다보니
- 어차피 100*100의 도화지이고, 숫자가 거대하지 않기 때문에 완전탐색을 진행하기로 결정했다.
- 해당 방식으로 둘레를 재던 중 0,0과 같이 가장 끝 라인에 색종이가 있을 경우 둘레를 잴 수 없는 문제가 발생해 해당 문제를 해결하여 정답을 맞추었다.

### 🤔 후기
- 배열의 가장 끝 가장자리를 생각하지 않고 구현을 했더니 계속해서 틀렸었다. 
- 처음에는 배열의 범위 안을 탐색하기 위해 범위 밖으로 넘어가면 해당 좌표를 아예 버림 처리를 해버렸는데 그러면 0,0 라인과 같이 가장 끝에 색종이가 붙어있을 경우 둘레를 잴 수 없는 문제가 발생했다. 
- 해당 경우 무조건 둘레에 해당되기 때문에 ans를 더해주는것으로 해결했다.
- 다시한번 문제 조건을 확인하는 습관이 필요하다는것을 깨달음
