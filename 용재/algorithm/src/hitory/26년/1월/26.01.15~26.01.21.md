# 🧩 후기 문제

## [도영이가 만든 맛있는 음식](https://www.acmicpc.net/problem/2961)

### 💡 아이디어 / 접근법
- 음식을 고르는 경우와 고르지 않는 경우로 계산을 진행했다.
여기서 문제점이 음식을 하나도 고르지 않는 경우가 발생하는데 그렇다면 diff가 1이 나오기 때문에
가장 작은 차이가 나올 확률이 굉장이 높아진다.
- 해당 경우를 제외하기 위해 sour(신맛), bitter(쓴맛)이 depth = n까지 갔을 때에도 초기값과 같다면
- 아무 재료를 사용하지 않은것이기에 케이스에서 제외함으로써 문제를 해결했다.
### 🤔 후기
- 재귀를 머리속으로 생각하면서 예외케이스를 상정하는것도 어려운데 아무재료를 사용하지 않는 경우가 나와
- 해당 경우를 어떻게 없앨지 고민하는데 시간을 잡아먹었다.

## [부등호](https://www.acmicpc.net/problem/2529)

### 💡 아이디어 / 접근법
- 처음에는 now라는 현재 찾는 순번을 나타내는 변수를 만들어서 깊이와 now를 통해 숫자를 탐색하고 리스트에 depth번째에 조건을 만족하는 수를 넣어 문제를 푸는 방식을 선택했다.
- 해당 방식은 가능한 모든 경우의 수를 출력하는 문제가 발생해 해당 숫자들을 배열에 담는 방식도 선택했다.
- 배열에 담아보니 021같이 0이 시작인 숫자가 문제가되어 숫자가 아닌 문자로 비교를 하려했으나 - '0'이 쉽사리 떠오르지 않아 처음부터 생각했다.
- 조건을 만족하는 숫자를 처음부터 함수의 매개변수로 문자의 형태로 넘기고 문자에 - '0'을 통해 비교하는 방식을 사용하면
- 리스트에 담아 크기를 비교하기도 쉽고, 코드 자체도 더 깔끔해져 해당 방식을 채택했다.(이전 방식은 함수 길이가 조건문으로 인해 엄청나게 길었음)
- 또 해당방식을 선택하면서 check 함수를 따로 만들어 사용하여 지저분했던 cal 함수를 정리할 수 있었다.
### 🤔 후기
- 아 좀 어려웠다. 배열을 만들고 해당 배열을 출력하는게 끝이 아니라
조건에 맞는 최대, 최소값을 출력하려면 조건에 해당하는 수만큼 순회를 해야했는데
- 여태 하던 방식처럼 result[i] 값을 순회해 출력하면 모든 조건에 부합하는 수가 출력되어 문제가 발생했다.
- 그 문제를 해결하는데 굉장히 많은 시간을 쏟아부었다.
