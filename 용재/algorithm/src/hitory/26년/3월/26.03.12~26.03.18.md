# 🧩 후기 문제

## [줄세우기](https://www.acmicpc.net/problem/2252)

### 💡 아이디어 / 접근법
- 위상정렬의 기본 문제라 위상정렬을 구현해 풀이했다.
- 위상정렬
  - 위상정렬의 구현 방식은 노드에 진입차수가 0인것 부터 큐에 넣은뒤
  - 해당 노드의 진출차수를 모두 없앤다.
  - 그럼 또 다른 노드의 진입차수가 0이 되는게 있다면 해당 노드를 큐에 넣는 방식을 반복한다
  - 진입차수가 0인 노드가 남지않았다면 사이클이 형성되어있는것이다.
### 🤔 후기
- [위상정렬 구현 - 자바](https://bcp0109.tistory.com/21), [위상정렬 개념](https://velog.io/@kimdukbae/%EC%9C%84%EC%83%81-%EC%A0%95%EB%A0%AC-Topological-Sorting)
- 두 사이트를 참고하여 위상정렬을 공부 후 문제풀이를 하였다.

## [게임 개발](https://www.acmicpc.net/problem/1516)

### 💡 아이디어 / 접근법
- 건물을 '동시에' 지을 수 있다고 해 dp를 적용했음
- 일단 건물은 모두 자기 자신을 짓는 시간을 가지기 때문에 시작 건물을 체크하면서 dp에 자기 자신의 시간을 넣었음
- 이후 건물이 지어지면서 dp의 next의 완료 시간과 now 시간 + next 건물의 순수 건축시간을 비교하여 갱신하는 식으로 문제 풀이를 하였음
### 🤔 후기
- dp가 섞이면서 좀 어려웠다. 처음엔 대소비교 필요없이 dp[now]+times[next]를 넣어주면 된다고 생각해 해당 방식으로 구현했으나
- 꼭 1~2개정도씩 맞지않는 결과가 나와 방법을 찾아보았고 해당 방식으로 풀었음
