# 🧩 후기 문제

## [유기농 배추](https://www.acmicpc.net/problem/1012)

### 💡 아이디어 / 접근법
- 이어진 배추는 배추 흰지렁이가 1마리만 있으면 되기 때문에 상하좌우만 탐색
- 상하좌우 탐색하면서 두번 탐색하지않게 숫자 바꿔주고 cnt를 늘린다
- 마지막에 cnt 출력해서 해결
### 🤔 후기
- 오랜만에 델타탐색이라 배열 범위를 벗어나는 설정하는것 부터 좀 해맸음
- 함수를 굳이 쓸 필요 없었으나 익숙해지기 위해 사용

## [알고리즘 수업 - 깊이 우선 탐색 1](https://www.acmicpc.net/problem/24479)

### 💡 아이디어 / 접근법
- dfs하면 stack 사용이 최근에 머리에 박혀서 처음에는 stack으로 풀려고 했었음
- 근데 어차피 인접 노드로 이동하기 때문에 스택을 굳이 쓸 필요 없다는것을 깨닫고 수정
- 노드와 간선을 사용하는 그래프를 볼때 배열속에 배열을 넣으면 편하다고 생각했음
- 자바의 경우 ArrayList로 해결하여 문제풀이 성공

### 🤔 후기
- dfs 기초중의 기초문제라 예전엔 쉽게 풀었던것 같은데 요즘 헤이해진게 체감이 확실히 온 문제
- dfs 구현을 하면서 꽤 많이 절었고 시간도 오래걸렸다. 결국 함수 구현 마지막의 마지막에 ai의 도움을 받음..

## [미로 탐색](https://www.acmicpc.net/problem/2178)

### 💡 아이디어 / 접근법
- 처음에는 bfs를 떠올리지 않고 델타로만 해결하려 했음
- 그런데 구현하고보니 한칸 전진 - 다음칸 탐색인데 완전 델타 탐색은 그저 [0,0],[0,1] 이런식으로 진행됨을 깨달음
- bfs구현의 필요성을 느껴 bfs 함수 구현후 풀이 성공

### 🤔 후기
- bfs 구현의 기초문제. visited 함수까진 쓸 필요가 없었던 문제지만 일부러 사용하여 문제를 풀었음

## 이번주 리뷰
- 델타탐색, bfs, dfs는 삼성SW 기준 IM 난이도인데도 꽤 오래걸렸다.
- 머리속에 어떤식으로 움직여야하는지는 떠올랐지만 그걸 코드로 적는데 고생을 했음
- 함수를 의도적으로 더 사용하면서 느끼는건데 어디까지 static 변수로 선언해 main 함수와 알고리즘 함수에서 사용해야할 지 고민이 된다.
- 이번주의 경우 유기농 배추의 경우 함수의 매개변수로 input 값을 넘겨줬지만 bfs의 경우 외부 static 변수로 선언한 후 재할당 하는 방식으로 진행을 했는데
- 1번의 실행내에 테스트 케이스가 많았더라면 무조건 매개변수로 줘야겠지만 1개의 테스트 케이스라면 고민이 되는 문제다.
